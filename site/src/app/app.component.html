Being able to configure and update your environments is an important part of DevOps. Doing these tasks quickly and easily is even more important. Ansible helps solve the problem of manually configuring servers one-by-one. Using Ansible as an automation tool System Administrators can update and configure servers from a single machine. This course is a way to learn how to setup Ansible and a sandbox environment to learn in. You’ll learn how to manage servers using Ansible playbooks, stand up a new environments, and monitor your cloud infrastructure.

See source code

1. Course Introduction

Discuss what Ansible is and some of its use cases. We will learn about why, when, and where it is used and similar tools used in the industry today.

2. Setup Enviornment

We will learn how to install Ansible and other tools get your system ready for hand on labs and experience you’ll use in this course.

3. Tasks

Learn about adhoc commands and how to use the Ansible command line tools. Gain experience with running our first Ansible commands.

4. Playbooks

Hands on experience with writing and running playbooks to achieve deployment, automation and orchestration.

5. Using Varibles

Learn how to refactor playbooks by using variables and making our automation more modular.

6. After Thoughts

We will go over common issues, testing playbooks, the future of automation as code and cleaning up our environment.
